# next-i18next-sample

A minimal **Next.js (Pages Router) + [next-i18next](https://github.com/i18next/next-i18next)** demo app ‚Äî purpose-built as the target repository for the **[LingoPilot](https://github.com/heyitschien/lingopilot-engine)** automated localization workflow.

> **What this demonstrates:** Push a change to any English locale file ‚Üí the LingoPilot GitHub App automatically opens a PR with pseudo-translated strings and Puppeteer screenshots of every route ‚Äî no human intervention required.

## LingoPilot Integration

This repo has the **LingoPilot GitHub App** installed. The automation runs as follows:

```
1. You edit any file under public/locales/en/
2. Push to main
3. LingoPilot bot (Probot) receives the webhook ‚Üí enqueues a job
4. LingoPilot worker dequeues ‚Üí clones repo ‚Üí generates pseudo-translations
5. Worker opens a PR: updates public/locales/zz-pseudo/ + commits screenshots
6. Dashboard at lingopilot.app shows the run with screenshot gallery
```

The `screenshots/` folder in this repo contains images committed by the bot from the most recent run.

### Bot-generated PR example

Each automated PR includes:
- Updated `public/locales/zz-pseudo/*.json` files
- Puppeteer screenshots at 1280√ó800 for each route

## Triggering a Run Manually

If you have the LingoPilot engine cloned locally:

```bash
# Via GitHub repository_dispatch (no commit needed)
gh api repos/heyitschien/next-i18next-sample/dispatches \
  --method POST \
  -f event_type=localization \
  -f "client_payload[sha]=main"

# Or just edit any source string and push
echo " " >> public/locales/en/home.json && git add . && git commit -m "test: trigger lingo bot" && git push
```
